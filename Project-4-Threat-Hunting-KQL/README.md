# Project 4 – Threat Hunting & Security Investigations using KQL

## Overview

This project demonstrates proactive threat hunting and security investigations using Microsoft Defender XDR Advanced Hunting (KQL), Windows Event Logs, and MITRE ATT&CK methodologies.

The objective was to identify suspicious activities, investigate indicators of compromise (IOCs), validate security alerts, and improve detection capabilities within a SOC environment.

## Technologies Used

- Microsoft Defender XDR
- Advanced Hunting (KQL)
- Windows Event Logs
- Microsoft Defender for Endpoint
- MITRE ATT&CK Framework
- Windows 11
- Windows Server 2022

## Key Activities

- Conducted proactive threat hunting using KQL queries
- Investigated suspicious PowerShell activity
- Analyzed authentication events and sign-in anomalies
- Identified Indicators of Compromise (IOCs)
- Validated security alerts and incident data
- Investigated process execution and parent-child process relationships
- Mapped findings to MITRE ATT&CK techniques
- Documented findings and remediation recommendations

## Sample Threat Hunting Scenarios

### Suspicious PowerShell Activity

Objective:
Identify potentially malicious PowerShell execution that may indicate attacker activity.

### Failed Login Investigation

Objective:
Detect brute-force attempts and authentication anomalies through log analysis.

### Process Execution Analysis

Objective:
Investigate suspicious process chains and identify abnormal system behavior.

### Endpoint Investigation

Objective:
Review endpoint telemetry and correlate security events across multiple data sources.
